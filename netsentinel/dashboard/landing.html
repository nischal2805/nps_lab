Workflow

From Target to Intelligence

A four-phase pipeline turns any codebase or live host into actionable security data.

1
Configure Target

GitHub URL, local filesystem path, or live hostname. NetSentinel validates reachability before scanning.

2
Analyze Code

Static analyzer extracts ports, API routes, hardcoded secrets, and SBOM dependency vulnerabilities.

3
Live Probing

Four concurrent probe modules sweep the target: network, TLS, HTTP headers, and DNS records.

4
Score & Report

CVSS 3.1 scores computed, domain grades assigned, OWASP mapped, dashboard launched automatically.
